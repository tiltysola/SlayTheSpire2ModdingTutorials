## Context

本项目是《杀戮尖塔2》模组制作教程，包含完整的入门到进阶知识体系。游戏基于 Godot 4.5.1 Mono 引擎开发，模组通过 DLL (C# 代码) + PCK (资源) + JSON (配置) 三件套形式加载。教程涵盖：

- **基础教程**：环境配置、代码修改、BaseLib 接口使用
- **进阶内容**：新人物添加、变量描述系统、先古对话
- **视觉资源**：卡图替换、Spine动画、风格原画绘制
- **版本迁移**：不同游戏版本间的兼容性处理

当前需求是将这些教程知识整合为 Qoder Skill，使 AI 能够在模组开发任务中自动应用最佳实践。

## Goals / Non-Goals

**Goals:**
- 创建结构清晰、内容全面的 skill 文件
- 覆盖模组开发全生命周期的知识点
- 提供可直接使用的代码模板和模式
- 包含常见问题的解决方案

**Non-Goals:**
- 不替代原有教程文档
- 不包含游戏玩法相关内容
- 不处理非 STS2 的游戏模组开发

## Decisions

### 1. Skill 文件结构设计

**决策**：采用分层结构组织 skill 内容
- 基础配置层：环境、依赖、项目结构
- 核心开发层：BaseLib 接口、Harmony 补丁
- 资源制作层：场景节点、本地化、视觉资源
- 问题解决层：调试方法、版本迁移

**理由**：分层结构便于 AI 根据上下文定位相关知识，避免信息过载

### 2. 代码模板格式

**决策**：使用完整可运行的代码示例，而非片段
- 每个代码块包含必要的 using 语句
- 注释说明关键参数的含义
- 提供类名和命名空间作为参考

**理由**：完整代码减少 AI 生成时的猜测，提高准确率

### 3. Skill 触发条件

**决策**：skill 在以下场景触发
- 用户提及"杀戮尖塔2"、"STS2"、"模组"等关键词
- 检测到项目中存在 BaseLib 依赖
- 用户请求添加卡牌、遗物、人物等游戏内容

**理由**：精准触发避免不相关场景的干扰

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 游戏版本更新导致 skill 内容过时 | 在 skill 中标注适用版本，提醒用户查阅最新教程 |
| Skill 内容过于冗长影响 AI 上下文 | 使用结构化标题和折叠建议，AI 可按需提取 |
| BaseLib 版本变化导致接口不兼容 | 说明 BaseLib 版本要求，提供版本检查建议 |
